# 일류 손익 대시보드 — 사용 가이드

---

## 1. 개요

**일류기획 손익 대시보드**는 매출·매입·순익 현황을 한눈에 확인할 수 있는 내부용 웹 대시보드입니다.

- **접속 주소**: `https://illryu-dashboard.vercel.app`
- **접속 방법**: 이메일 + 비밀번호 로그인 (계정은 관리자가 발급)

---

## 2. 데이터 구조

```
구글 시트 (당월 실시간 데이터)
        ↕ 마감 버튼 클릭 시
Supabase DB (과거 마감된 데이터 영구 보관)
        ↕
대시보드 (화면 표시)
```

| 구분 | 데이터 위치 | 조회 방법 |
|------|------------|----------|
| **당월 데이터** | 구글 시트 | 📡 구글 시트 실시간 버튼 |
| **과거 마감 데이터** | Supabase DB | 년/월 필터로 조회 |

---

## 3. 월별 업무 흐름

```
① 매월 초  : 구글 시트에 당월 데이터 입력 시작
② 매월 중  : 📡 실시간 버튼으로 수시 확인
③ 매월 말  : 🗓️ 이번 달 마감 버튼 클릭
              → 시트 데이터가 DB에 영구 저장
              → 다음 달 시트 새로 작성 시작
④ 다음 달  : 이전 월은 DB에서 언제든 조회 가능
```

---

## 4. 화면 구성

### 상단 헤더 버튼

| 버튼 | 기능 |
|------|------|
| 📡 구글 시트 실시간 | 현재 구글 시트 데이터를 즉시 불러와 화면에 표시 |
| 📊 DB 보기로 전환 | 실시간 모드 종료, DB 데이터로 복귀 |
| 💾 보고서 저장 | 현재 필터 조건 + 집계 요약을 DB에 기록 |
| 🗓️ 이번 달 마감 | 구글 시트 데이터 → DB 영구 저장 (월 마감) |
| 🔄 새로고침 | 데이터 다시 불러오기 |
| 📄 PDF 다운로드 | 현재 화면을 PDF로 저장 |

---

### 필터바

| 필터 | 설명 |
|------|------|
| **년도** | 전체 / 2024 / 2025 / 2026 |
| **월** | 전체 / 1~12월 |
| **주차** | 전체 / 1~5주차 (한국 기준 월요일 시작) |
| **일** | 전체 / 1~31일 |
| **구분** | 접수형 / 관리형 / 보장형 / 테스트 / 보장완료 / 중단건 (다중 선택 가능) |
| **검색** | 업체명 / 상호명 / 담당자 부분 검색 |

> 로그인하면 **현재 한국 날짜 기준 당월**이 자동으로 설정됩니다.

> 필터를 변경하면 **상단 KPI 카드, 모든 차트, 리스트, PDF** 전부 동일 기준으로 연동됩니다.

---

### 대시보드 섹션

| 섹션 | 내용 |
|------|------|
| **KPI 카드 (4개)** | 매출 합계 / 매입 합계 / 순익 합계 / 손익률 + 전월 대비 증감 |
| **주차별 순익** | 주차별 막대 차트 (보라=흑자, 빨강=적자) |
| **업체별 순익** | 상위 15개 가로 막대 차트 + 전체보기 모달 |
| **일자별 손익 추이** | 매출/매입/순익 꺾은선 차트 |
| **구분별 손익 요약** | 접수형·관리형 등 구분별 집계 테이블 |
| **적자 업체 TOP 5** | 손실 큰 업체 순위 |
| **업체별 상호명 분포** | 대행사별 상호명 순익 분포 피벗 테이블 |
| **상세 거래 리스트** | 전체 거래 내역 (정렬·검색·수정·추가 가능) |

## 5. 거래 수정 / 추가

### 기존 거래 수정
1. 상세 거래 리스트에서 **수정할 행 클릭**
2. 팝업에서 날짜 / 담당자 / 업체명 / 구분 / 매출 / 매입 수정
3. 순익은 자동 계산 (매출 - 매입)
4. **저장** 클릭 → 즉시 반영

### 누락 거래 추가
1. 상세 거래 리스트 우측 상단 **`+ 거래 추가`** 버튼 클릭
2. 정보 입력 후 **저장**

### 구분 일괄 변경
1. 변경할 행 **체크박스 선택** (복수 선택 가능)
2. 상단에 나타나는 입력란에 **새 구분명** 입력
3. **✓ 변경 적용** 클릭

## 6. 기술 구성 (관리자용)

| 항목 | 내용 |
|------|------|
| **프론트엔드** | Next.js (Vercel 배포) |
| **데이터베이스** | Supabase (PostgreSQL) |
| **인증** | Supabase Auth (이메일/비밀번호) |
| **Google 연동** | Google Sheets API (Service Account) |
| **소스코드** | GitHub `work-min/illryu-dashboard` |

### 환경변수 (Vercel)

| 변수명 | 설명 |
|--------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 프로젝트 주소 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 공개 키 |
| `GOOGLE_SPREADSHEET_ID` | 구글 시트 ID |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | Google 인증 키 (base64 인코딩) |

## 7. 계정 관리

### 새 계정 추가
1. [supabase.com](https://supabase.com) 접속 → 프로젝트 선택
2. 좌측 메뉴 **Authentication** → **Users**
3. **Add User** → 이메일 / 비밀번호 입력
4. **Email Confirm** 체크 후 저장

### 코드 배포 방법
GitHub `main` 브랜치에 push하면 Vercel이 자동으로 배포합니다.
수동 배포: Vercel 대시보드 → Deployments → Redeploy

---

## 8. 구글 시트 연동 주의사항

- 구글 시트의 **탭 이름**은 반드시 `리스트`, `귀속 입금건` 으로 유지
- 시트 공유: `illryu-sheets@illryu-dashboard.iam.gserviceaccount.com` 에 **뷰어 권한** 필요
- 시트의 컬럼 구조(순서/이름)가 바뀌면 데이터 파싱 오류 발생 가능

---

## 9. 문제 해결

| 증상 | 원인 | 해결 방법 |
|------|------|----------|
| 로그인이 안 됨 | 계정 없음 또는 비밀번호 오류 | 관리자에게 계정 발급 요청 |
| 실시간 데이터가 안 불러와짐 | 구글 시트 공유 설정 문제 | 시트 공유 목록에 서비스 계정 이메일 확인 |
| 데이터가 비어 있음 | 필터 조건이 너무 좁음 | 초기화 버튼 클릭 |
| 마감 후 데이터가 0건 | 구글 시트에 데이터 없음 | 시트 내용 확인 후 재시도 |

# 일류기획 대시보드 — 기능 요약 (현재 반영 상태)

> 최종 업데이트: 2026-07-08

---

## 1. 손익 대시보드 (`/dashboard`)

| 기능 | 내용 |
|------|------|
| KPI 4개 | 매출 / 매입 / 순익 / 손익률 + **전월 동일 구간 대비 증감** |
| 전월 비교 기준 | 현재 필터(주차/일/상품 구분/검색)와 **동일 조건**으로 전월 데이터 조회 후 비교 |
| 차트 | 일자별 손익 추이 / 접수형 순익 TOP15 + 전체보기 모달 / 주차별 순익 막대 |
| 상품별 요약 | 접수형·관리형·보장형 등 상품 구분별 집계 테이블, 미분류 발생 시 거래 목록 표시 |
| 적자 TOP5 | 손실 큰 업체 순위 |
| 보장형 · 관리형 손익 분포 | 대행사명 → 상호명 기준 피벗 테이블, `전체/관리형/보장형` 탭 조회 |
| 상세 거래 리스트 | 정렬 / 컬럼 필터 / 검색 / 체크박스 일괄 변경(상품 구분·대행사명·상호명) / 행 클릭 수정 / 신규 추가 |
| 필터 | 년도·월·주차(1~5)·일(1~31)·상품 구분 다중선택·검색(대행사명/상호명/담당자) — 전부 서버사이드 |
| 기타 | 다크모드, PDF 다운로드, 구글 시트 동기화, 월 마감(시트→DB 확정) |

### 현재 레이아웃 순서

1. KPI 카드: 총 매출 / 총 매입 / 영업 이익 / 손익률
2. 상품별 손익 요약 | 일자별 손익 추이
3. 접수형 순익 (상위 15개) | 적자 업체 TOP 5
4. 주차별 순익
5. 보장형 · 관리형 손익 분포
6. 상세 거래 리스트

### 상품 구분 기준

- 구글 시트 컬럼명은 `상품 구분`을 우선 사용합니다.
- 기존 시트 호환을 위해 `구분` 컬럼도 fallback으로 지원합니다.
- 접수형 순익 차트는 `상품 구분 = 접수형` 데이터만 대행사명 기준으로 합산합니다.
- 보장형 · 관리형 손익 분포 탭 기준:
  - 전체: 관리형, 보장형, 보장 완료, 중단건
  - 관리형: 관리형
  - 보장형: 보장형, 보장 완료, 중단건

## 2. 정산 대시보드 (`/settlement`)

| 기능 | 내용 |
|------|------|
| KPI 4개 | 총 영업이익 / **총 급여** / 총 지출 / 최종 이익 (+ 전월 대비) |
| 비율 KPI 3개 | 인건비 비율 / 지출 비율 / 최종 이익률 — **분모를 총매출 기준으로 계산** (2026-06-08 변경, 기존 영업이익 기준에서 변경) |
| 당월 데이터 | 구글 시트(`급여(지급액)`, `지출` 탭)에서 실시간 조회 |
| 과거 월 데이터 | DB(`settlement_records`)에서 조회, 없으면 간소화 입력 폼 (영업팀 급여 / 정규직 급여 / 고정·유동 지출) |
| 수정 모드 | 저장된 데이터 인라인 수정 → PATCH로 최종이익 자동 재계산 |
| 월별 추이 차트 | 영업이익 vs 최종이익 라인 차트 (저장된 전체 월) |

## 3. 일마감 (`/daily-report`)

| 기능 | 내용 |
|------|------|
| 이체 내역 | 엑셀 붙여넣기(금액+내용 자동 분리) 또는 직접 입력, 합계 자동 계산 |
| 미수금 / 미지급금 | 구글 시트(`미수현황` 탭)에서 자동 불러오기 또는 직접 추가 |
| 특이사항 | 자유 텍스트 입력 |
| 보고서 미리보기 | 입력 즉시 텍스트 보고서 자동 생성 → 클립보드 복사 |
| 저장/불러오기 | 날짜별로 `daily_reports` 테이블에 upsert, 날짜 변경 시 자동 로드 |

## API 라우트

| 경로 | 역할 |
|------|------|
| `/api/sheets` | 거래 데이터 동기화 (리스트 + 귀속 입금건 탭) |
| `/api/settlement` | 당월 급여·지출 (구글 시트) |
| `/api/settlement-record` | `settlement_records` CRUD |
| `/api/daily-report` | `daily_reports` GET/POST (upsert) |
| `/api/daily-report/sheets` | 미수현황 탭에서 미수/미지급 파싱 |

## ⚠️ 확인 필요 (DB 스키마 갱신 필요)

- **`daily_reports` 테이블이 `supabase/migrations/`에 없음.**
  코드 기준 추정 컬럼: `date(date, UNIQUE)`, `transfers jsonb`, `receivables jsonb`, `payables jsonb`, `notes text`, `created_at`, `updated_at`
  → Supabase에서 실제 컬럼/RLS 확인 후 `001_tables.sql`, `002_rls.sql`에 추가 필요

## 최근 반영 이력 (최신순)

1. 보장형 · 관리형 손익 분포에 `전체/관리형/보장형` 탭 추가
2. 구글 시트 `상품 구분` 컬럼명 우선 파싱, 기존 `구분` fallback 유지
3. 미분류 거래 목록 표시 추가
4. 상세 거래 리스트 일괄 변경 대상 확장: 상품 구분 / 대행사명 / 상호명
5. 접수형 순익 차트: 접수형 데이터만 대행사명 기준 상위 15개 표시
6. 대시보드 레이아웃 재배치 및 명칭 변경
